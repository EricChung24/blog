OpenClaw 是什麼？那隻紅色龍蝦是你的私人 AI 助理

跑在你自己電腦上的開源 AI Agent，會收信、寫 code、控制瀏覽器、接 LINE，還能自己寫擴充套件。Peter Steinberger 做的這個工具，讓「擁有自己的 AI 助理」不再是夢話。

Apr 21, 2026
閱讀時間: 5 分鐘

如果 Manus 是跑在別人家雲端的 AI 員工，那 OpenClaw 就是住在你自己電腦裡的那一個。

Logo 是一隻紅色龍蝦，但別被可愛外表騙了——這其實是目前最有潛力的本地端 AI Agent 之一。

OpenClaw 是什麼？

一句話：

「一個跑在你自己電腦上的開源 AI 助理，會自己做事、有記憶、能接通訊軟體，還能自己寫擴充套件。」

作者是 Peter Steinberger（iOS 開發圈的老朋友，以前做 PSPDFKit），在 2025 年推出的個人專案。

跟主流 AI 產品最大的差異：

項目ChatGPT / Claude / ManusOpenClaw
跑在哪別人家雲端你自己電腦
資料上傳到對方伺服器全程留在本地
擴充官方提供自己寫 / 社群共享
費用訂閱費開源免費（只付 API 費用）
整合度有限想接什麼就接什麼

為什麼這個東西值得注意？

因為它解決了一個很多人有的焦慮：

「我想要 AI 幫我做事，但我不想把所有資料都給 OpenAI / Anthropic。」

OpenClaw 的設計哲學很明確：Privacy-first。

  • 跑在你自己的 Mac / Windows / Linux 上
  • 記憶資料存在本機
  • 要不要呼叫雲端 AI（Claude、GPT）由你決定
  • 也可以接本地 LLM（Ollama 之類）完全離線

等於是把「AI 助理」這件事從雲端服務變成一個你擁有的程式。

它能做什麼？

官方列出 50+ 整合，但真正厲害的是它能自己組合這些整合。

幾個常見情境：

情境 1：訊息助理

接上 LINE / Telegram / WhatsApp / Slack / iMessage / Signal / Discord，你可以直接傳訊息給它：

  • 「幫我整理今天的 GitHub 通知」
  • 「搜一下這個餐廳的評價」
  • 「提醒我下午 3 點開會」

不用開電腦、不用開瀏覽器，手機傳訊息就完事。

情境 2：Email 處理

接 Gmail 之後，可以說：

  • 「把今天所有未讀信分類，重要的摘要給我」
  • 「回覆這封信，語氣要客氣但堅定」
  • 「把這封信的附件存到 Obsidian 的 2026-04 資料夾」

情境 3：coding 助手

  • 「幫我把這個 repo clone 下來，跑測試看看哪裡壞了」
  • 「查一下昨天 push 的 commit 有沒有影響 production」
  • 「把這段程式重寫成 TypeScript」

情境 4：生活雜事

  • 「訂今晚 7 點的餐廳」
  • 「Spotify 放點 jazz」
  • 「把今天的支出記進我的財務 Google Sheet」

情境 5：自我擴充

OpenClaw 最狂的一點：它可以自己寫擴充套件。

你說：「我想要一個能自動抓 Hacker News 前 10 名摘要的功能。」

它會：

  1. 寫一個新的 skill（擴充模組）
  2. 測試它
  3. 加進自己的能力清單

AI 幫 AI 自己擴充能力，這個遞迴很有意思。

補充：Claude 本身也有 Agent 功能

在講跟 Manus 的差異之前，值得提一下：

Anthropic 的 Claude 本身也有 Agent 能力，叫做 Claude Computer Use（電腦使用）以及整合在 Claude.ai 的 Projects + Tools 功能。

Claude 可以：

  • 控制瀏覽器、操作視窗（Computer Use）
  • 呼叫工具（搜尋網路、執行程式碼、讀取檔案）
  • 在 Projects 裡保存長期記憶與文件

跟 OpenClaw 的差別是：Claude 的 agent 能力是雲端服務、官方整合，而 OpenClaw 是把 Claude 當成「大腦」，在你的電腦上自主執行。

也就是說，OpenClaw + Claude API = 把 Claude 的智慧裝進一個跑在你家的機器人。

跟 Manus 有什麼不同？

前一篇才介紹完 Manus，很多人會問：「這兩個差在哪？」

項目ManusOpenClaw
執行位置雲端 VM本機
是否開源❌ 封閉✅ 開源
訊息介面只有網頁7+ 通訊軟體
記憶單次任務長期累積
擴充官方主導任何人可寫
門檻註冊就能用要自己安裝

講白一點：

  • Manus 像一個你租的雲端員工——好用、高階、但要付錢、資料給對方
  • OpenClaw 像一個你自己養的助手——要花時間調教、但完全是自己的

怎麼安裝？

OpenClaw 是開源的，安裝流程大致是：

1. 安裝主程式

Mac / Windows / Linux 都有安裝包，到 openclaw.ai 下載。

2. 提供 API Key

你要有一個 LLM 能力來源，可選：

  • Anthropic API（Claude）— 推薦，能力最強
  • OpenAI API（GPT）
  • Ollama（本地模型，完全離線）

3. 授權要接的服務

想接 Gmail 就 OAuth 授權 Gmail，想接 LINE 就做 LINE bot 綁定。

4. 開始對話

裝好後第一次跟它講話，它會記住你的偏好。

比如你說：「我叫 Eric，平常用 Notion 做筆記，習慣早上 6 點起床。」

之後它都記得。

使用的幾個注意事項

1. 它是個 alpha 階段的工具

OpenClaw 還很新，會有 bug、會有奇怪行為。

不建議用在絕對不能出錯的工作流（例如付款、發正式信件給客戶）。

2. 記憶會累積成包袱

記憶越多，它越懂你，但也可能把舊的錯誤偏好一直帶著。

建議定期檢查它的記憶檔、刪掉過時的內容。

3. 本地執行不代表完全安全

即使資料在本地，它呼叫 Claude / GPT 時還是會把 context 送出去。

如果你真的想完全離線，搭配 Ollama 跑本地模型，但能力會差很多。

4. 擴充能力 = 雙面刃

它能自己寫 script、跑 shell 指令——這代表它也有能力弄壞你的系統。

建議：

  • 重要操作先看一眼它要做什麼再放行
  • 不要給它 root 權限
  • 敏感資料夾設 read-only

誰適合用？

很適合：

  • 重視隱私：不想把所有資料給雲端服務的人
  • 工程師：能自己寫擴充、修 bug、調整流程
  • 重度自動化愛好者：喜歡把生活各種雜事自動化
  • 愛好 open source：支持開源生態的人

暫時不適合：

  • 完全不會裝軟體的人：入門門檻還是比 ChatGPT 高
  • 要 100% 穩定的商業應用：還在快速迭代，不夠穩
  • 只想聊天：對話體驗不如 ChatGPT 專精

為什麼是「龍蝦」？

這個很多人問。

OpenClaw 的 Logo 是一隻紅色龍蝦——「Claw」就是龍蝦的螯。

這隱喻大概是：有抓取、有操作、有個性，不是軟綿綿的助手。

我覺得這個 branding 很成功，至少一眼就記得。

小結

2025 到 2026 年，AI Agent 的發展進到一個分岔路口：

  • 雲端路線：Manus、Claude Computer Use、各種 SaaS 代理
  • 本地路線：OpenClaw、open-interpreter、各種自建 agent

兩條路各有優缺點，但**「AI Agent 應該長在自己電腦裡」**這個理念，會吸引一群重視主導權的使用者。

如果你：

  • 已經用過 ChatGPT / Claude，想更進一步
  • 對資料隱私有要求
  • 願意花點時間裝設定
  • 想有一個真正「屬於自己」的 AI 助理

那這隻紅色龍蝦值得一試：

👉 openclaw.ai

未來我們可能每個人都會有一個像這樣的東西，陪你處理大大小小的雜事。OpenClaw 讓這個未來提早到了。
